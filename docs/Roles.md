# Roles

> Источник: TZ_MarketTJ_ClaudeCode.md, разделы 4, 5.

## Структура интерфейса: 3 части + Courier mini-interface

Интерфейс — **не 4 полноценные панели**, а:

1. **Public/Customer Website** — Customer работает внутри публичного сайта, отдельной dashboard-панели у него нет;
2. **Farmer Panel** — полноценная панель;
3. **Admin Panel** — полноценная панель;
4. **Courier mini-interface** — 2 страницы (My Deliveries, Delivery Details), НЕ считается отдельной полноценной панелью.

Маршруты и страницы — см. [Frontend.md](Frontend.md).

## Admin

Администратор управляет всей системой.

Возможности:

* вход в административную панель;
* просмотр пользователей;
* блокировка и разблокировка пользователей;
* подтверждение или отклонение фермеров;
* управление категориями;
* просмотр объявлений;
* скрытие неподходящих объявлений;
* просмотр всех заказов;
* назначение курьера (с учётом доступности `IsAvailable`, региона и типа транспорта — Farmer курьера не выбирает);
* просмотр доставок;
* просмотр отзывов;
* просмотр общей статистики.

## Farmer

Фермер размещает и продаёт продукты.

Возможности:

* регистрация;
* вход в систему;
* заполнение профиля;
* отправка заявки на подтверждение;
* создание объявления;
* загрузка изображений;
* изменение цены;
* изменение доступного количества;
* временное скрытие объявления;
* просмотр заказов;
* принятие заказа;
* отклонение заказа;
* изменение статуса на «Готов к выдаче» (ReadyForPickup);
* просмотр отзывов;
* просмотр статистики продаж.

Фермер не может публиковать объявления, пока администратор его не подтвердил. Фермер **не выбирает курьера самостоятельно** — это делает только Admin.

## Customer

Покупатель ищет и заказывает продукты. Customer работает внутри **Public/Customer Website** — отдельной dashboard-панели у Customer нет.

Возможности:

* регистрация;
* вход;
* просмотр каталога;
* поиск продуктов;
* фильтрация;
* просмотр информации о фермере;
* добавление товара в корзину;
* изменение количества в корзине;
* оформление заказа;
* указание адреса доставки;
* просмотр истории заказов;
* отмена заказа до его принятия фермером;
* подтверждение получения;
* создание отзыва.

## Courier (mini-interface, не полноценная панель)

Курьер доставляет заказ. Для Courier **нет отдельной полноценной панели** — только небольшой mini-interface из 2 страниц (My Deliveries, Delivery Details, см. [Frontend.md](Frontend.md)).

Возможности (ограниченный набор, только это):

* вход в систему;
* просмотр списка назначенных ему доставок (`My Deliveries`);
* просмотр деталей доставки (`Delivery Details`): адрес фермера, адрес покупателя, телефон фермера, телефон покупателя;
* изменение статуса: `PickedUp` → `InDelivery` → `Delivered`;
* просмотр истории доставок.

Курьер не выбирает себе доставки и не назначается сам — назначение выполняет только Admin.

## Типы клиентов

Для покупателя рекомендуется использовать тип:

```text
Retail — розничный покупатель
Wholesale — оптовый покупатель
```

В MVP тип клиента можно сохранить, но отдельную сложную бизнес-логику для опта не реализовывать.
