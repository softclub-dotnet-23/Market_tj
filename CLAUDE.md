# Market.tj — память проекта

Полная спецификация: `TZ_MarketTJ_ClaudeCode.md` в корне репозитория. Прочитай её целиком перед началом любой сессии, если ещё не читал в этом контексте.

## Автоматический лог прогресса (обязательно, каждую сессию, без напоминаний)

После завершения **любого** законченного действия — сущность реализована, endpoint готов, баг исправлен, миграция применена, тест написан и т.д. — сразу, без просьбы пользователя:

1. Определи сегодняшнюю дату в формате `YYYY-MM-DD`.
2. Открой (или создай, если не существует) файл `backend/progress/YYYY-MM-DD.md`.
3. Допиши в конец файла запись по шаблону:

   ```markdown
   ## HH:MM — <короткое название сделанного>
   - Что сделано: <1-3 предложения>
   - Файлы: <изменённые/созданные файлы>
   - Решения/примечания: <если были нетривиальные решения — иначе "—">
   ```

4. Делай это сам, автоматически, каждый раз — не спрашивай разрешения и не жди отдельной просьбы.

**Не путать с:**
- `docs/PROGRESS.md` — общий чек-лист по этапам проекта (разделы 26–27 ТЗ), обновляется реже, вручную по завершении крупных пунктов.
- `docs/phase-summaries/PhaseN-summary.md` — итоговый отчёт по завершении целого Phase (раздел 37.2 ТЗ), пишется по подробному шаблону (What Was Built, Architecture Decisions и т.д.).
- `backend/progress/YYYY-MM-DD.md` — самый мелкий уровень: лог каждого отдельного законченного действия внутри дня, ведётся непрерывно и автоматически.

## Прочие обязательные правила проекта

- Id и все внешние ключи — `int`, не `Guid` (раздел 18 ТЗ).
- SOLID в Application/Infrastructure (раздел 11.2.2 ТЗ) — сервисы зависят от интерфейсов, не от конкретных реализаций.
- Redis-кэш через `ICacheService`/`RedisCacheService` (раздел 18 ТЗ) — не обращаться к `StackExchange.Redis` напрямую из Application.
- Коммиты по схеме `Phase{N} Step {M} [BE|FE|FULL]: описание` (раздел 37.1 ТЗ).
- Frontend — React (не Blazor), 3 части + Courier mini-interface, не 4 полноценные панели (раздел 4.5, 14 ТЗ).
- Не добавлять функциональность вне MVP/согласованного списка сущностей без явного запроса пользователя.
