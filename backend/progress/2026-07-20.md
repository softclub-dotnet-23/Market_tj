## 15:10 — Login/Register: анимация "растущей яблони" в зелёной панели (AuthPanel)

- Что сделано: заменено фото фермера с корзиной в `AuthPanel` (используется и `Login`, и `Register`) на PNG-изображение яблони с прозрачным фоном (пользователь сохранил его сам в `assets/photos/image.png`, переименовано в `tree.png`). Дерево "вырастает" при монтировании (~1.8s, `scaleY` 0.15→1 от нижнего края + `clip-path: inset()` снизу вверх, ease-out-quart), после чего (`onAnimationComplete`) на нём по очереди появляются 7 отдельных CSS-яблок (spring bounce, шаг ~0.15s), каждое с лёгким бесконечным покачиванием. Яблоки — не картинки, а сгенерированные div'ы с radial-gradient (блик сверху-слева → тёмно-красный снизу-справа), inset box-shadow для объёма, градиентным черешком и листиком. `prefers-reduced-motion` — читается через `window.matchMedia` (как в `LanguageContext`/`SmoothScroll`) один раз на модуль; при reduce=true дерево и все яблоки рендерятся сразу в финальном состоянии без анимаций.
- Файлы: `Frontend/src/components/layout/AuthPanel.tsx`, `Frontend/src/assets/photos/index.ts` (добавлен экспорт `treePhoto`), `Frontend/src/assets/photos/tree.png` (переименован из `image.png`), `Frontend/src/locales/ru/layout.json` и `Frontend/src/locales/tj/layout.json` (обновлён `authPanel.heroAlt` под новую картинку).
- Решения/примечания:
  - Ссылку на Claude Design (claude.ai/design/p/...) прочитать через WebFetch не удалось (нет сетевого доступа из sandbox к этому хосту) — реализовано строго по текстовому ТЗ из промпта пользователя, без визуальной сверки с дизайн-проектом.
  - Исходная PNG уже содержит яблоки, запечённые в самом фото (это фото дерева с уже видимыми плодами), а не "голое" дерево. Раздельная поштучная CSS-анимация яблок поверх картинки — компромисс: реальные яблоки с фото видны сразу по мере роста (появляются снизу вверх вместе с кроной через clip-path), а 7 дополнительных объёмных CSS-яблок с bounce/покачиванием накладываются поверх кроны как отдельный анимационный слой поверх фото, что и даёт визуально запрошенный эффект "яблоки появляются по очереди после того, как дерево выросло". Точные координаты яблок (`top/left` в %) подобраны на глаз по кроне дерева на фото — при желании пользователь может попросить подвинуть.
  - Убраны floating-превью товаров (`floaters`) вокруг фото — они были частью старой композиции с фото-корзиной и не сочетались тематически с деревом; не заменялись на аналог.
  - Проверено: `tsc -b --noEmit` — 0 ошибок; `oxlint` — чисто; `vite build` — собирается успешно, `tree.png` (773 KB) корректно попадает в бандл. Визуально в браузере не проверялось — в текущей среде нет инструмента для рендеринга/скриншота фронтенда; `npm run dev` в этой sandbox-среде может выглядеть падающим — известная особенность окружения, не связанная с этим изменением (см. память).

## 15:40 — AuthPanel: реализация по исходнику из Claude Design ("Login Tree Animation.dc.html")

- Что сделано: пользователь дал прямую ссылку на проект Claude Design (`891069ef-eb93-42ae-94af-d19150e5c0b3`, файл `Login Tree Animation.dc.html`) и попросил реализовать именно её. Прочитал файл и ассеты через MCP-инструмент `claude_design` (`get_project`/`list_files`/`get_file`) — в проекте оказался отдельный спрайт `assets/apple.png` (33×39, реалистичное яблоко с прозрачным фоном) и 9 точных координат яблок (`left/top` в %, `size` в px) в JS-компоненте файла. Переписал предыдущую реализацию (7 hand-drawn CSS-градиентных яблок из шага 15:10) на точное воспроизведение дизайна:
  - `treeGrow`: `scaleY` 0.05→1, `scaleX` 0.4→1, `opacity` 0.4→1 (с явным промежуточным кадром на 30% длительности), `clip-path: inset()` снизу вверх — `cubic-bezier(0.22, 1, 0.36, 1)`, 1.8s (точные значения из `@keyframes treeGrow` дизайна, было `cubic-bezier(0.25,1,0.5,1)` без scaleX/opacity-кадра).
  - Яблоки — теперь `<img src={applePhoto}>` (реальный спрайт из дизайна), не CSS-градиенты; 9 позиций 1-в-1 из `spots` дизайна.
  - `appleGrow`: bounce-кадры `scale(0)→1.15→0.95→1` на `0%/60%/80%/100%` (точные проценты из `@keyframes appleGrow`), стаггер по 0.15s (эта часть — из исходного текстового ТЗ пользователя, т.к. в самом `.dc.html` JS-компонент считает `spots`/`swayDelay`, но фактическую привязку `apple-grow`/stagger-задержку к разметке дизайнер ещё не успел доделать — класс `.apple-grow` объявлен только в `prefers-reduced-motion` media query, без базового правила и без вычисляемого delay в `renderVals()`).
  - `appleSway`: покачивание `rotate(-3deg)↔rotate(3deg)`, по умолчанию 4s (проп `swaySpeed` дизайна), стаггер 0.6s на яблоко — точно из `@keyframes appleSway` и JS (`swayDur`/`swayDelay`).
  - Добавлена пасхалка из дизайна: клик по яблоку → `appleFall` (`translateY(420px) rotate(220deg)`, fade out, `cubic-bezier(0.5,0,0.85,0.4)`, 1.15s) — яблоко падает и остаётся "упавшим" (локальный state компонента), точные значения взяты из `@keyframes appleFall`/`.apple-fall` дизайна.
